# Кредитный скоринг  
Задача — предсказать вероятность дефолта клиента по кредиту на основе истории его кредитных продуктов.

**Данные** Модели обучены начасти данных, каждая запись — один кредитный продукт.  
Целевая переменная — `flag` (1 — дефолт, 0 — нет).

**Метрика** — ROC‑AUC. Лучший результат на валидации — **0.7563**, на тесте платформа показала **0.7355**.

Я пошла по пути **агрегации признаков на уровне клиента**, поскольку каждый клиент имеет несколько записей. Основные шаги:

1. **Агрегация**  
   - One‑hot кодирование всех признаков (кроме `id` и `rn`) и суммирование по клиенту.  
   - Добавление статистик: `mean`, `max`, `min` по всем числовым колонкам.  
   - Добавление `product_count` (число кредитов) и `rn_max`/`rn_min`.  
   - Ранговые признаки: `log_id`, `id_rank`, `product_count_rank`, `rn_span_ratio` (чтобы уловить временные тренды).

2. **Моделирование**  
   - **XGBoost** — настройка гиперпараметров с помощью Optuna (60 итераций).  
   - **CatBoost** — подбор параметров руками (дал лучший индивидуальный результат — 0.7558).  
   - **LightGBM** — пробовала, но он показал низкое качество (0.719), поэтому исключён из финального ансамбля.

3. **Ансамбль**  
   - Взвешенный бленд CatBoost (0.95) и XGBoost (0.05).  
   - Это дало небольшое улучшение до **0.7563** на валидации.

4. **Работа с памятью**  
   - Google Colab имеет ограничение ~12 ГБ RAM, поэтому я читала тестовые данные по частям (батчами по 250 000 строк) с помощью `pyarrow.parquet`.  
   - При агрегации следила, чтобы последний `id` в батче не разрывался, перенося его в следующий чанк.


## Результаты

| Модель | AUC (валидация) |
|--------|----------------|
| CatBoost (индивидуально) | 0.7558 |
| Ансамбль (0.95 Cat + 0.05 XGB) | **0.7563** |
| XGBoost (индивидуально) | 0.7529 |
